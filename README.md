# CS-230-Software-Testing

How can I ensure that my code, program, or software is functional and secure?

I make sure my code works by testing it a lot, especially using unit tests to catch mistakes early. I also check edge cases so the program does not break with weird inputs. For security, I focus on validating inputs and making sure only the correct data is accepted so nothing unexpected causes issues.

How do I interpret user needs and incorporate them into a program?

I start by reading the requirements carefully and breaking them down into what the program actually needs to do. Then I turn those into smaller parts in my code. I try to think from the user perspective and make sure the program does what they expect, and I use tests to confirm everything works the way it should.

How do I approach designing software?
I usually plan things out first before coding so I do not get lost halfway through. I think about how different parts of the program connect and try to keep everything simple and organized. I also design it in a way that makes it easy to test and fix later if needed.
